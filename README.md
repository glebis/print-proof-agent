# PrintProof — агент-корректор для типографии

> Вычитка буклетов, листовок и финальной вёрстки за день вместо недели. Агент работает автономно — менеджер только проверяет.

Пилотное приложение на **Claude Agent SDK**, построенное в [Claude Code Lab](https://agency-lab.glebkalinin.com) (когорта 5) как сквозной пример: от JTBD-интервью и аудита архитектуры до работающего приложения с 83 юнит-тестами, проверенного в браузере.

## Задача (Job to Be Done)

Менеджер типографии: срочные заказы упираются в этап корректуры (~неделя при цели «день») — поиск свободного корректора, ожидание правок, переговоры о сроках и цене через почту, мессенджеры и Excel, которые «не сходятся».

> «Когда заказ доходит до корректуры, я хочу получить вычитанный текст без уговоров и переговоров с людьми — чтобы это занимало день вместо недели, а срок и цена были известны заранее».

**Главный UX-принцип:** приложение оптимизировано под человека-**проверяющего**, а не под соавторство. Каждая правка несёт provenance — какой модуль её сделал, на каком основании, с какой уверенностью. Каждое действие объясняет последствия до клика. Ошибка в тираже необратима, поэтому ничего не уходит в печать без явной приёмки менеджером.

## Как это работает: каскад B-lite

Архитектура выбрана после внешнего аудита (вместо «оркестратор + параллельные субагенты по категориям» — категории живут в данных и UI, а не в runtime):

```
Загрузка (DOCX / TXT / MD / PDF / PNG / JPEG)
  ↓ нормализация в immutable-текст rev.1
Оценка — объём, время, цена сразу (предсказуемость = продукт)
  ↓
┌─ Rule-pass    детерминированно, 0 токенов: тире, кавычки-ёлочки, пробелы, словарь клиента
├─ LLM-pass     один query() Agent SDK: орфография, пунктуация, стиль → Edit[]
└─ Vision-pass  только PDF/картинки: налезающие блоки, читаемость → задание дизайнеру
  ↓ Resolver (код): дедуп, конфликты по span, сортировка
Приёмка менеджером — поштучно / категорией / все; хоткеи; word-diff; фильтры; поиск
  ↓ только accepted, сборка курсором по immutable-базе
Экспорт: Markdown · DOCX · протокол приёмки · CSV · задание дизайнеру · архив (JSON)
```

### Ключевые инженерные решения

- **Целостность правок.** Все span-координаты — в неизменяемом тексте одной ревизии; экспорт собирает результат курсором по отсортированным принятым правкам, а не применяет их по одной с пересчётом offsets (классический баг текстовых редакторов; принцип как в CRDT/git).
- **LLM не считает координаты.** llm-pass возвращает фрагмент + контекст, позицию вычисляет код; галлюцинированные фрагменты отбрасываются.
- **snap-to-text для PDF.** Vision оценивает bbox «на глаз» (±10%); для PDF рамки уточняются по точной геометрии текстового слоя — каждое замечание ложится на своё слово.
- **Подписка, не API-ключ.** В пилоте Agent SDK работает под подпиской Claude Code (`ANTHROPIC_API_KEY` снят перед вызовами).
- **Тестируемая архитектура.** Бизнес-логика — чистые функции в `src/lib/`, покрытые юнит-тестами (мс, без браузера и LLM); интеграция в роуты/UI остаётся тонкой.

## Возможности

| Область | Что есть |
|---|---|
| **Корректура** | rule-pass (типографика + словари), llm-pass (орфография/пунктуация/стиль), vision-pass (макеты) |
| **Приёмка** | provenance-тултипы, word-diff, навигация ‹ ›, мигание цели, хоткеи (j/k/a/r, RU-раскладка), фильтры по статусу/категории, поиск, сортировка, предпросмотр итогового текста |
| **Макеты** | многостраничные PDF, bounding-boxes на изображении, клик-по-рамке → замечание, быстрое принять/отклонить |
| **Профили клиентов** | детерминированные правила + стилевой промпт + few-shot примеры; редактирование через чат-агент (Vercel AI SDK + умные виджеты) и напрямую формой; самообучение словаря из повторяющихся правок |
| **Автоматизация** | авто-принятие безопасных правок (opt-in, только rule-pass/profile, не «важные»), перепроверка с сохранением решений, очередь проверок |
| **Контроль и аудит** | протокол приёмки (кто/что/когда), фактический расход токенов и стоимость, оценка vs факт, SLA-бейдж, детектор дубликатов, валидация загрузки |
| **Экспорт** | Markdown, DOCX, протокол, CSV (Excel-ru), задание дизайнеру, полный архив заказа (JSON) |
| **Доступ** | ACL: admin / manager / client / designer; админка пользователей и клиентов |

## Запуск

```bash
# зависимости
npm install

# Agent SDK под подпиской Claude Code — снять API-ключ
unset ANTHROPIC_API_KEY

# дев-сервер (порт 3742)
npm run dev

# тесты
npm test
```

Откройте http://localhost:3742. В шапке — демо-переключатель пользователя (роли). Загрузите файл, дождитесь каскада, примите правки, экспортируйте.

> **Пилот, не продакшн.** Зафиксированные ограничения: DOCX-экспорт без сохранения оригинальной вёрстки; vision до 10 страниц PDF; демо-аутентификация без паролей; хранение файлов на диске.

## Дорожная карта до продакшна

Приложение прошло внешний аудит реализации (Codex). Закрытые по итогам баги: object-level ACL, защита приёмки от чужих правок, fail-closed при ошибке Agent SDK. Осталось до реального потока заказов в типографии:

- **Print preflight** (детерминированный) — PDF/X, размер страницы, bleed/safe area, DPI, RGB/CMYK, встроенные шрифты, overset text. Vision-pass не заменяет prepress-проверку.
- **Durable-очередь** — атомарный claim задания в SQLite (`UPDATE … RETURNING` / отдельная таблица jobs), lease/lock, retry/backoff, cancellation, heartbeat воркера. Текущий `spawn('npx tsx')` — пилотный.
- **Версионный workflow** — комментарии к правкам, назначение дизайнеру/менеджеру, финальное approve клиента, история ревизий, diff между прогонами.
- **Эксплуатация** — деплой, настоящая аутентификация (пароли/magic-link), S3-хранилище, переход с подписки на API-ключ.
- **Расширения корректуры** — OCR бумажных правок корректора, точный DOCX-патчинг с сохранением вёрстки.
